I used VS Code for three years. it was comfortable, had great extensions, and just worked. So why did i switch to Neovim? The honest answer is — speed and control.

The Problem with VS Code

VS Code is a great editor but it's an Electron app, which means it's essentially a browser running your editor. On older machines or large projects, this shows. Startup time, memory usage, and occasional sluggishness were things i just accepted — until i didn't.

My First Week with Neovim

Brutal. i won't sugarcoat it. i spent more time configuring than coding. But something clicked around day five — i stopped reaching for the mouse. My hands stayed on the keyboard and i started moving through code faster than i ever had before.

What i Use Now

i run LazyVim as my base config. it gives me LSP support, fuzzy finding, and git integration out of the box. From there i've added a few plugins for my specific workflow.

Should You Switch?

Probably not if you're just starting out. VS Code is genuinely excellent and the learning curve of Neovim is real. But if you've been coding for a while and want more control over your environment, it's worth trying for a week.
